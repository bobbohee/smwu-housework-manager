# 우리집 살림 매니저 — 설계 문서

## 개요

함께 사는 사람들이 집안일 당번을 확인하고 기록하는 반응형 웹앱.

- 대상: 함께 거주하는 2~N명 (룸메이트, 가족, 셰어하우스 등). 특정 가정에 고정되지 않은 **범용 앱**.
- 멤버 이름·집안일 항목·색상은 모두 그룹이 직접 정의한다. 앱에 하드코딩된 가정-특화 값은 없다.
- 기존 방식: 캘린더 앱에 수동으로 "OO 설거지", "OO 빨래" 등 기록 → 이를 전용 웹앱으로 대체.
- 본 문서의 데이터 예시에 나오는 "언니/보희/동생", "우리집" 등은 **설명용 예시**일 뿐 고정값이 아니다.

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React + Next.js (App Router) |
| 백엔드 | Next.js API Routes (최소한만 사용) |
| 인증 | Firebase Auth (이메일/비밀번호) |
| 데이터베이스 | Firebase Firestore |

- 프론트엔드 중심 아키텍처: 클라이언트에서 Firestore SDK로 직접 읽기/쓰기
- API Route는 서버 검증이 꼭 필요한 경우에만 사용
- Firestore Security Rules로 데이터 접근 권한 제어

## 인증 & 사용자

- Firebase Auth에는 이메일/비밀번호만 저장
- 사용자 이름은 회원가입 시 입력 (모든 그룹에서 동일한 이름 사용)
- users 컬렉션에 uid, name 저장

## 주요 기능 (7가지)

### 1. 그룹 관리
- 그룹(우리집) 생성 → 생성자가 방장(owner)
- 초대 코드 생성 → 공유 → 코드로 멤버 합류
- 초대 코드는 유효 기간/사용 제한 없이 언제든 사용 가능
- 멤버 목록 관리 (방장만 강퇴 가능)
- 방장 위임 — 방장이 다른 멤버에게 방장 권한을 넘길 수 있음
- 다중 그룹 지원 — 한 계정이 여러 그룹에 가입 가능
- 홈 상단에서 현재 그룹명 탭 → 그룹 전환/선택

### 2. 집안일 관리

**프리셋 & 동적 추가**
- 그룹 생성 직후 범용 프리셋을 제시한다(체크박스로 선택 → 한 번에 생성). 프리셋 후보:
  - 설거지(순번제), 빨래(순번제), 거실 청소(순번제), 방 청소(순번제), 밥하기(순번제)
  - 음식물쓰레기 비우기(고정제), 일반쓰레기 배출(고정제), 화장실 청소(고정제)
- 각 프리셋은 권장 모드·기본 색상을 가지지만, **선택 시 그룹의 chores 컬렉션에 복사본으로 생성**된다(참조 아님). 따라서 그룹마다 자유롭게 이름·모드·색상·순서·규칙을 수정하거나 삭제할 수 있다.
- 프리셋을 0개 선택(빈 화면에서 시작)하는 것도 허용한다.
- 프리셋은 시작 템플릿일 뿐이며, 이후 "집안일 관리" 화면에서 언제든 추가/수정/삭제가 가능하다(동적 추가).

집안일 등록 시 두 가지 모드 중 선택:
- 등록 시 색상 팔레트(아래 "색상 팔레트" 절의 10색)에서 색상 선택 → 캘린더에서 구분용

**순번제**
- 설거지, 빨래, 밥, 청소 등
- 집안일별로 참여 멤버 설정 가능 (예: 화장실 청소는 언니, 동생만)
- 집안일별로 순서 설정 가능 (예: 설거지는 언니→보희→동생, 빨래는 동생→보희→언니)
- 설정된 순서대로 돌아감
- 완료 버튼 1번 = 차례 1번 소진, 즉시 다음 사람으로 넘어감
- 주기가 정해져 있지 않고, 집안일이 쌓이면 차례인 사람이 수행

**고정제**
- 쓰레기 배출, 화장실 청소 등
- 담당자별 스케줄 지정. 두 가지 방식 지원:
  - 요일 지정 (예: 언니=월, 보희=수, 동생=금)
  - 주기 지정 (예: 화장실 청소 — 언니: 2주마다, 시작일 3/31)
- 완료 버튼 없음, 기록 저장 안 함
- 해당 날짜에 홈에서 알림만 표시 (예: "오늘 화장실 청소 담당: 언니")

### 3. 완료 규칙
- 집안일별 규칙을 등록 (예: 설거지 → "음식물쓰레기 비우기")
- 규칙을 지키지 않으면 완료로 인정하지 않는 약속
- 앱에서는 참고용으로 표시 (강제 체크 아님)

### 4. 완료 체크 (순번제만 해당)
- "완료" 버튼 한 번으로 처리
- 완료 시 자동으로 다음 사람으로 넘어감: `currentTurnIndex = (currentTurnIndex + 1) % rotationOrder.length`
- 완료 기록이 choreLog에 저장됨

**완료 버튼 권한**
- 기본: 순번제 chore의 완료 버튼은 **현재 차례 멤버(`rotationOrder[currentTurnIndex]`)에게만** 활성화된다. 다른 멤버에게는 비활성/숨김.
- chore에 `allowProxyComplete: boolean`(기본 `false`) 필드를 둔다. `true`이면 참여 멤버(`rotationOrder`에 포함된) 누구나 대신 완료할 수 있다 — 차례 멤버 부재 시 커버 용도.
- 대신 완료 시 통계 귀속과 감사 기록을 분리한다:
  - `completedBy` = **차례였던 멤버 uid**. 차례 소진 및 통계 집계의 귀속 대상.
  - `completedByActual` = **실제 버튼을 누른 멤버 uid**. 본인이 직접 완료한 경우 `completedBy`와 동일.
- 즉 `allowProxyComplete: false`이면 항상 `completedBy == completedByActual`이다.

### 4-1. 완료 내역 비활성화
- 규칙을 지키지 않은 완료 기록을 방장이 비활성화 처리 가능
- 비활성화 시 사유 입력 필수
- 비활성화된 내역은 삭제되지 않고 비활성 상태로 유지 (모든 멤버가 확인 가능)
- 비활성화된 기록은 통계 집계에서 제외
- 순번제의 경우 비활성화 시 currentTurnIndex를 되돌림 (차례 복원)

### 5. 꽝뽑기 (랜덤 돌리기)
- 돌발 집안일(엄마 심부름 등)에 대해 담당자를 랜덤 배정
- 그룹 멤버 전원 표시, 체크박스로 참여/제외 토글
- 당첨 인원 수 조정 가능 (기본 1명)
- "꽝뽑기" 버튼 클릭 → 참여 멤버 중 N명 랜덤 선택
- 결과: 폭탄 아이콘 + 당첨자 이름 표시
- 결과 확정 후 choreLog에 기록

### 6. 캘린더 뷰
- **구현 방식: 앱 내 자체 캘린더.** Google Calendar 등 외부 캘린더 API에 의존하지 않는다. Firestore `choreLog`의 `completedAt`을 기준으로 월별 그리드를 직접 렌더링한다(외부 연동/내보내기는 현재 범위 밖, 추후 확장 가능).
- 월별 캘린더에서 날짜별로 완료된 집안일 표시 (순번제·꽝뽑기 기록)
- 집안일별 색상으로 구분
- 날짜별 "설거지: 보희" 형태로 표시
- 클릭 시 상세 정보 팝업 (완료 시간, 담당자 등)
- 기존 캘린더 앱 사용 방식과 유사한 UX

### 7. 히스토리/통계
- 이번 달 멤버별 완료 횟수
- 집안일별 완료 횟수
- 숫자(테이블) + 차트(막대/파이) 둘 다 표시

## 페이지 구성 (7개)

| 페이지 | 핵심 기능 |
|--------|----------|
| 로그인/회원가입 | 이메일/비밀번호 로그인 |
| 홈 (대시보드) | 상단 그룹 전환 + 집안일별 현재 담당자 확인 + 완료 버튼 + 고정제 알림 + 규칙 확인 |
| 캘린더 | 월별 캘린더에서 날짜별 완료 기록 (색상 구분), 클릭 시 상세 팝업 |
| 집안일 관리 | 추가/수정/삭제, 모드 설정(순번제/고정제), 색상 선택, 규칙 등록 |
| 꽝뽑기 | 멤버 선택, 인원 조정, 폭탄 아이콘으로 당첨자 표시 |
| 통계 | 멤버별/집안일별 완료 횟수 (숫자 + 차트) |
| 그룹 설정 | 멤버 목록, 강퇴(방장만), 초대 코드 확인/공유 |

## 사이트맵 (3클릭 내 도달)

```
로그인 → 홈(1) → 캘린더(2) → 상세 팝업(3)
              → 집안일 관리(2) → 집안일 상세/편집(3)
              → 꽝뽑기(2) → 결과(3)
              → 통계(2)
              → 그룹 설정(2)
```

하단 탭 네비게이션으로 홈/캘린더/꽝뽑기/통계/설정에 바로 접근 가능 → 대부분 2클릭 내 도달.

## 데이터 구조 (Firestore)

컬렉션 4개:

### users
```json
{
  "uid": "...",
  "name": "보희",
  "email": "bohee@example.com",
  "groupIds": ["groupId_1", "groupId_2"]
}
```

### groups
```json
{
  "name": "우리집",
  "inviteCode": "ABC123",
  "ownerId": "uid_of_creator",
  "memberUids": ["uid_언니", "uid_보희", "uid_동생"]
}
```

### chores
```json
{
  "groupId": "...",
  "name": "설거지",
  "mode": "rotation | fixed",
  "color": "#4A90D9",
  "rotationOrder": ["uid_언니", "uid_보희", "uid_동생"],
  "currentTurnIndex": 1,
  "allowProxyComplete": false,
  "fixedSchedule": [
    { "uid": "uid_언니", "type": "weekly", "days": ["mon"] },
    { "uid": "uid_보희", "type": "weekly", "days": ["wed"] },
    { "uid": "uid_동생", "type": "interval", "intervalDays": 14, "startDate": "2026-03-31" }
  ],
  "rules": ["음식물쓰레기 비우기", "싱크대 물기 닦기"]
}
```

- `mode: "rotation"` → 순번제. `rotationOrder`로 참여 멤버 및 순서 지정, `currentTurnIndex`로 현재 차례 관리.
- `allowProxyComplete` → 순번제 전용. `false`(기본)면 현재 차례 멤버만 완료 버튼 활성. `true`면 참여 멤버 누구나 대신 완료 가능.
- `mode: "fixed"` → 고정제. `fixedSchedule`로 담당자별 스케줄 지정. `type: "weekly"`는 요일 지정, `type: "interval"`은 N일 주기 지정. 완료 기록 없음.
- `color` → 캘린더에서 집안일 구분용 색상 (팔레트에서 선택).
- `rules` → 완료 규칙 목록 (참고용).

### choreLog
```json
{
  "choreId": "...",
  "groupId": "...",
  "completedBy": "uid_보희",
  "completedByActual": "uid_보희",
  "completedAt": "2026-03-31T14:00:00",
  "type": "rotation | random",
  "active": true,
  "deactivatedBy": null,
  "deactivatedAt": null,
  "deactivateReason": null
}
```

- `type: "rotation"` → 순번제 완료 기록
- `type: "random"` → 꽝뽑기 결과 기록
- `completedBy` → 차례 소진·통계 귀속 대상 uid. `completedByActual` → 실제 버튼을 누른 uid(대신 완료 시 둘이 다를 수 있음). 통계 집계는 `completedBy` 기준.
- `active` → 기본 true. 방장이 비활성화 시 false로 변경.
- `deactivatedBy`, `deactivatedAt`, `deactivateReason` → 비활성화 시 기록. 모든 멤버가 사유 확인 가능.
- 캘린더 뷰: `completedAt` 기준으로 날짜별 조회 (비활성화 내역도 표시, 구분 가능하게)
- 통계: 월별로 `completedBy` 집계 (active=true인 것만)
- 순번 넘기기: 완료 처리 시 해당 chore의 `currentTurnIndex`를 +1

## 구현 세부 규칙

### 색상 팔레트 (10색)
집안일 등록 시 아래 고정 팔레트에서 1색 선택. 캘린더/리스트 구분용.

```
#4A90D9 (블루)   #E74C3C (레드)    #2ECC71 (그린)   #F39C12 (오렌지)  #9B59B6 (퍼플)
#1ABC9C (틸)     #E91E8C (핑크)    #34495E (네이비)  #95A5A6 (그레이)  #F1C40F (옐로우)
```

### 초대 코드 생성 규칙
- 영문 대문자 + 숫자 6자 (예: `ABC123`). 혼동되는 문자(`0/O`, `1/I/L`)는 제외.
- 그룹당 1개 유지. 생성 시 기존 코드와 충돌하면 재생성한다.
- 유효 기간·사용 횟수 제한 없음(언제든 사용 가능).

### 고정제 "오늘 담당" 판정
홈에서 고정제 알림은 다음 규칙으로 계산한다(완료 기록 없음, 표시 전용).
- `type: "weekly"`: 오늘 요일이 `days[]`에 포함되면 해당 `uid`가 오늘 담당.
- `type: "interval"`: `floor((오늘 - startDate) / 1일)`이 0 이상이고 `intervalDays`로 나누어떨어지면 오늘 담당. (즉 `startDate`부터 `intervalDays` 간격마다 표시)
- 같은 chore에 여러 담당자가 같은 날 걸릴 수 있으며, 모두 나란히 표시한다.
